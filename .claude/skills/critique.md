# Skill: Writing Critique Output

## Purpose
Generate sharp, useful creative feedback. Not generic AI praise.

## Tone Model
Think: senior creative director in a review. Direct. Specific. Useful.

- BAD: "The typography is clean and modern."
- GOOD: "Body copy drops below 14px on mobile — legible on desktop, fails on small screens."

- BAD: "Great use of white space!"
- GOOD: "Breathing room is fine but the CTA is visually buried — it reads as tertiary, not primary."

## Structure for Each Insight
1. What the issue is (label — short)
2. Why it matters (detail — one sentence)
3. How confident we are (high/medium/low)

## Fix Next Format
Max 3 actions. Each must be:
- Specific (not "improve typography", but "increase body text to minimum 16px")
- Actionable (something the user can do today)
- Ordered by impact

## Creative Director Notes Format
Four buckets only:
- **Weak**: things that are technically fine but feel undercooked
- **Generic**: things that could be from any brand
- **Off-brand**: things that contradict the brand (if guideline provided)
- **Performance risks**: things that will reduce effectiveness with the target audience

## What to Avoid
- Praise for meeting minimum standards
- Hedging language ("might", "could potentially", "in some cases")
- Restating what the asset is before critiquing it
- More than 3 items per Fix Next list
- Confidence: high unless you genuinely aren't sure
